# Список изменений

Все значимые изменения приложения 3X-UI Manager для Android.
Формат основан на [Keep a Changelog](https://keepachangelog.com/), версии — по
[семантическому версионированию](https://semver.org/).

🇬🇧 [English version](CHANGELOG.md)

## [0.5.4] — 2026-06-29

### Изменено
- **Один способ выйти из панели — «Выйти».** Отдельный пункт меню *Отключиться*
  убран. Вместо него у каждой панели в переключателе ⇄ (даже если сохранена
  всего одна) есть действие *Выйти*, которое забывает сохранённые URL и токен
  этой панели. Выход из последней панели возвращает на экран подключения — то
  есть делает всё то же, что *Отключиться*, но без двух разных кнопок.

## [0.5.3] — 2026-06-29

### Изменено
- **Страница «mTLS узлов» теперь полноэкранная.** Это был единственный пункт
  ⋮-меню, открывавшийся диалогом; теперь он в одном стиле с остальными экранами
  (DNS, Outbounds, Routing, …) — та же раскладка и цветовая гамма.

## [0.5.2] — 2026-06-29

### Исправлено
- **Компактная панель Outbounds.** Шапка больше не переносит заголовок
  «Outbounds»: «Сохранить» стало иконкой дискеты, а «Импорт из vless:// ссылки»
  убран в ⋮-меню (рядом с Импорт / Экспорт).

## [0.5.1] — 2026-06-29

### Добавлено
- **Несколько панелей — multi-instance.** Подключайтесь к нескольким панелям 3x-ui и
  переключайтесь между ними: кнопка ⇄ в верхней панели открывает переключатель,
  где можно сменить активную панель, добавить ещё одну (пустая форма подключения)
  или убрать. Панель, в которую вы уже были залогинены, переносится в список
  автоматически. «Отключиться» теперь забывает активную панель и переходит на
  другую сохранённую (или на экран подключения, если их больше нет). Активная
  панель показывается в верхней панели, а при переключении обновляются все экраны
  с данными.
- **Pull-to-refresh на экране узлов.** Свайп вниз обновляет статусы узлов — как на
  дашборде.

### Исправлено
- **Версия узла теперь обновляется после апдейта.** Самообновление узла качает
  новую сборку и перезапускается — это дольше одного обновления, поэтому версия
  узла раньше оставалась старой, пока не уйдёшь в дашборд и обратно. Теперь
  приложение опрашивает узел, пока его версия реально не сменится.
- **Замок приложения больше не мешает быстрому переключению.** Замок (код-пароль
  / отпечаток) теперь выжидает короткий grace-период перед повторной блокировкой,
  поэтому переключение в другое приложение (например, скопировать адрес панели) и
  обратно не спрашивает пароль; после разблокировки возвращает на тот экран, где
  вы были, а не на дашборд.

### Изменено
- **Диалоги обновления предупреждают о потере соединения.** При обновлении узла,
  обновлении панели или её перезапуске теперь выводится предупреждение: если
  приложение подключено к панели *через* неё, соединение прервётся и нужно будет
  переподключиться вручную.

## [0.4.2] — 2026-06-26

### Добавлено
- **Список клиентов: быстрый переключатель вкл/выкл.** В каждой строке
  клиента появился переключатель (как у inbound'ов) — включить или выключить
  клиента можно прямо в списке, не открывая его карточку.

### Исправлено
- **«Онлайн по серверам»: правильная привязка клиентов.** Диалог «Онлайн»
  больше не показывает клиента подключённым сразу к нескольким серверам.
  Теперь используется `POST /panel/api/clients/onlinesByGuid` (панель
  v3.4.0+): каждый online-email возвращается ровно под одним GUID сервера,
  через который реально проходит трафик. Для панелей старше v3.4.0 прежняя
  логика (по принадлежности inbound) остаётся как запасной вариант.
- **Высота поля поиска клиентов.** Поле «Поиск по email» растягивалось на
  две строки из-за длинного текста метки — теперь ограничено одной строкой.

### Изменено
- **Кнопки экспорта/импорта в Маршрутизации и DNS** заменены на иконки
  (↑ импорт / ↓ экспорт) вместо мелких текстовых ссылок: теперь заметнее
  и удобнее нажимать.

## [0.4.1] — 2026-06-26

### Добавлено
- **Inbounds: текущая скорость.** В списке inbound'ов показывается текущая
  скорость ↑/↓ (Б/с) по каждому inbound, опрашивается пока открыта вкладка
  (панель v3.4.0).
- **IP-лог клиента.** В листе клиента появилась кнопка *IP-лог*: показывает
  последние IP подключений клиента со временем и узлом (`@ node`), через который
  они прошли, плюс действие *Очистить* (панель v3.4.0).
- **Inbound: редактор stream settings (JSON).** Сворачиваемая секция «Расширенно:
  stream settings (JSON)» делает редактируемым весь объект `streamSettings`, так
  что новые поля 3.4.x, не вынесенные в структурную форму — Verify Peer Cert By
  Name, Curve Preferences, REALITY Limit Fallback, XHTTP Session ID Table/Length,
  пресет Real client IP в sockopt, FinalMask и пр. — можно задать вручную (как уже
  работает продвинутое у outbound).
- **Mutual TLS для узлов (mTLS).** В редакторе узла появился режим проверки TLS
  «mtls» (API-токен становится необязательным), а пункт **mTLS узлов** в меню ⋮
  позволяет скопировать CA этой панели (зарегистрировать на узле) и задать
  доверенный родительский CA (чтобы управляющая панель управляла этой как узлом).
  Панель v3.4.0.
- **Узел: подключение панель→узел через outbound.** В редакторе узла появился
  выбор «Outbound для подключения» (теги Xray-outbound, blackhole исключён;
  пусто = напрямую). Панель v3.4.0.
- **Экспорт/импорт клиентов + удаление несвязанных.** В меню ⋮ на странице
  Клиенты: экспорт всех клиентов в JSON (`{client, inboundIds}`, копировать/
  поделиться), импорт из такого JSON (существующие email пропускаются) и удаление
  клиентов без привязки к inbound. Панель v3.4.0.
- **Генератор ключа VLESS encryption.** В VLESS-inbound кнопка «Сгенерировать
  ключ VLESS encryption» тянет варианты ключей панели (X25519 / ML-KEM-768 ×
  native / xorpub / random), показывает пару decryption/encryption (с
  копированием) и умеет вставить decryption в настройки inbound (панель v3.4.1,
  xray-core v26.6.21+).
- **Массовые операции с клиентами.** Долгое нажатие на клиента включает режим
  выбора, дальше выбранных можно разом включить, выключить, изменить (добавить/
  убавить дни и трафик, задать или очистить XTLS-flow) или удалить (bulk-эндпоинты
  панели v3.4.1).
- **HTTP-outbound: кастомные заголовки.** В форму HTTP-outbound добавлен редактор
  заголовков (ключ/значение) для CONNECT-заголовков к вышестоящему прокси
  (`settings.headers`); раньше задать их было нельзя.
- **Обновление узлов на dev-канал.** В подтверждении обновления узла появился
  чекбокс «dev-сборка (последний коммит)» — с ним узел ставит роллинг
  dev-latest вместо стабильного релиза (панель v3.4.1).
- **Счётчики клиентов на дашборде.** Карточка вверху дашборда (перед CPU)
  показывает, сколько клиентов всего и сколько сейчас онлайн; по нажатию
  открывается список онлайна по серверам.
- **Вкл/выкл правил маршрутизации.** У каждого правила в редакторе Маршрутизации
  появился переключатель (как в панели 3.4.x): отключённое правило сохраняется,
  но затемняется и при сохранении вырезается панелью из рабочего конфига (нужна
  панель **v3.4.0+**; старые панели флаг игнорируют). Внутреннее правило
  статистики (`api`) остаётся всегда включённым.
- **Экспорт/импорт правил, outbounds и DNS-серверов.** Правила маршрутизации,
  outbounds и DNS-серверы можно выгрузить в JSON (скопировать или поделиться) и
  загрузить обратно, вставив JSON — голый массив, `{rules|outbounds|servers:[…]}`
  или обёртку `{routing|dns:…}`; импортированные записи добавляются к текущим.
- **Фильтр списка клиентов по статусу и группе.** Кнопка фильтра рядом с поиском
  (с бейджем числа активных фильтров) открывает лист — как Drawer-фильтр в самой
  панели — где список сужается по статусу (**Онлайн**, Активные, Выключенные,
  Исчерпанные) и по **группе** (используемые группы). Секции сужают независимо;
  «Сбросить всё» очищает фильтры.

### Изменено
- **Понятнее индикатор онлайна в списке клиентов.** Онлайн-клиенты теперь
  отмечаются ярко-зелёным кружком (офлайн — серым), чтобы сразу было видно, кто
  подключён — прежний синий кружок сливался с карточкой.
- **Редактор inbound предупреждает про зарезервированный API-порт.** Ввод порта
  62789 (внутренний Xray API по умолчанию) показывает предупреждение — панель
  отклоняет loopback-inbound на нём (v3.4.0).
- **Удаление узла блокируется, пока к нему привязаны inbound'ы.** Кнопка удаления
  в редакторе узла недоступна (с подсказкой), пока inbound'ы не отвязаны — как в
  панели, которая иначе отклоняет удаление (v3.4.0).
- **Список клиентов обновляется сам.** Пока открыта вкладка Клиенты, приложение
  опрашивает панель каждые несколько секунд — новые подключения, трафик и статус
  онлайн появляются без ручного обновления (панель v3.4.0).
- **WireGuard: убрано устаревшее поле Workers.** xray-core v26.6.22 его больше не
  использует — поле удалено из формы WireGuard-outbound (старые конфиги не
  затронуты, ключ просто игнорируется).
- **Интервалы истории метрик — как в панели (v3.4.x).** Выбор периода на графиках
  дашборда теперь 2m / 1h / 3h / 6h / 12h / 24h / 2d / 7d (история хранится до
  7 дней) вместо прежних 30m / 2h / 5h.
- **Аккуратный экран «Узлы» без узлов.** Текст «Нет узлов» теперь по центру, с
  иконкой и отступами, а не прижат к краю слева.
- **Группа клиента выбирается из существующих групп.** В редакторе клиента (и при
  создании, и при изменении) поле *Группа* стало выпадающим списком с
  автодополнением: показывает группы, которые уже используются у других клиентов,
  — теперь её выбираешь, а не вводишь вручную, так что опечатка больше не разобьёт
  одну группу на две. Ввести совсем новое название по-прежнему можно — новая
  группа создаётся прямо здесь.

## [0.3.23] — 2026-06-11

### Исправлено
- **Код-пароль больше не спрашивается сразу после ручного входа.** Если задан
  код-пароль приложения, вход на экране подключения (ввод URL + токена) тут же
  показывал запрос кода — бессмысленно, ведь ввод токена и есть аутентификация.
  Теперь замок включается только когда сохранённая сессия восстанавливается при
  запуске (вернувшийся пользователь) или когда приложение свёрнуто в залогиненном
  состоянии. Доработка к 0.3.22.

## [0.3.22] — 2026-06-11

### Исправлено
- **Блокировка приложения (код-пароль / отпечаток) больше не спрашивается, когда
  вы не залогинены.** Код-пароль защищает интерфейс панели, а не экран ввода
  токена — поэтому разлогиненное приложение (на экране входа после отключения или
  слетевшего токена) открывается сразу к полю токена, без запроса кода. Сворачивание
  экрана входа тоже больше не включает замок. Когда вы залогинены — сворачивание
  по-прежнему блокирует и запрашивает код при возврате, как и раньше. Сам код-пароль
  сохраняется (не сбрасывается при выходе) — он просто применяется только когда есть
  сессия, которую нужно защитить.

## [0.3.21] — 2026-06-11

### Изменено
- **Приложение теперь работает только по токену и требует панель v3.3.0 или
  новее.** Вход по логину и паролю (и 2FA) полностью убран — экран подключения
  спрашивает только URL панели и **API-токен**. На v3.3.0+ токен достаёт до всего
  управляющего API, поэтому логин больше ничего не даёт; его удаление убирает
  механику сессий/CSRF/cookie и периодическую переавторизацию (с повторным
  запросом 2FA). Существующий профиль с токеном продолжает работать; если вы
  раньше входили только по логину, создайте токен в панели:
  **Settings → Security → API Token**. Экран подключения теперь подсказывает, где
  его взять (сноска под полем токена).

## [0.3.20] — 2026-06-11

### Изменено
- **Недействительный токен / истёкшая сессия теперь возвращают на экран входа.**
  Если токен удалён или отключён в панели (или сессия по логину истекла и не
  продлевается), приложение больше не висит «подключённым», молча сыпля ошибками
  на каждый запрос — оно возвращается на экран подключения с понятным сообщением
  («API-токен больше не действителен…») и преднаполненными полями, чтобы вы ввели
  рабочий токен и переподключились. Запросы теперь шлют
  `X-Requested-With: XMLHttpRequest`, чтобы панель отвечала на отказ авторизации
  кодом **401** (а не 404), который приложение отличает от настоящего «не найдено».

## [0.3.19] — 2026-06-11

### Добавлено
- **Администрирование панели (меню ⋮):** управление самой панелью через
  token-доступный API настроек.
  - **Учётная запись админа** — смена логина и пароля входа в панель (для
    подтверждения нужны текущие данные; API-токен после смены продолжает работать).
  - **API-токены** — список, создание (плейнтекст нового токена показывается один
    раз для копирования), вкл/выкл и удаление.
  - **Перезапуск панели** — рестарт службы панели (с подтверждением; приложение
    переподключается через несколько секунд).

### Изменено
- Исправлены устаревшие надписи «с API-токеном это недоступно» по всему приложению
  и в README: на панели **v3.3.0+** токен покрывает редактор конфига Xray,
  настройки и ссылки подписки. Заглушка экрана Xray-конфига теперь нейтральная
  «не удалось загрузить» (показывается только при реальной ошибке загрузки), а
  подсказки про подписку отмечают, что база читается и по токену.

## [0.3.18] — 2026-06-11

### Добавлено
- **Бэкап / восстановление (меню ⋮):** скачать бэкап базы панели в файл и
  восстановить панель из него. Бэкап — это **вся база**: настройки, inbounds,
  клиенты и конфиг Xray, поэтому один файл сохраняет всё. Панель сама выбирает файл
  под свой движок (SQLite `x-ui.db` / PostgreSQL `x-ui.dump`) и принимает любой
  обратно — приложение не зависит от движка. Используется системный выбор файлов
  (Storage Access Framework); восстановление сначала спрашивает подтверждение и
  перезапускает Xray (кратко рвёт соединения). Работает по API-токену
  (`GET /panel/api/server/getDb`, `POST /panel/api/server/importDB`).

### Изменено
- Документация: уточнено, что на панели **v3.3.0+** API-токен покрывает и
  **настройки**, и **редактор конфига Xray**, и **авто-определение ссылки-подписки** —
  весь управляющий API переехал под `/panel/api/*`, который токен и авторизует.
  Раньше в README это значилось как «только сессия по логину».

## [0.3.17] — 2026-06-10

### Добавлено
- **Редактор outbounds:** структурный список outbounds в меню ⋮ (только при входе
  по логину, как и конфиг Xray) — добавить / править / удалить / переупорядочить
  (порядок = приоритет; индекс 0 — маршрут по умолчанию). Формы для vless, vmess,
  trojan, shadowsocks, socks, http, freedom, blackhole и wireguard, с транспортом и
  безопасностью (TCP / WS / gRPC / HTTPUpgrade / xHTTP / kcp, TLS / REALITY) для
  прокси-протоколов. **Импорт из `vless://`-ссылки** — вставьте ссылку, и появится
  готовый outbound. Нишевые типы (dns, loopback, WARP, NordVPN, TUN, reverse,
  outbound-подписки) правятся через поле сырого JSON. Любая правка сливается в
  полный конфиг, не затрагивая routing / balancers / DNS / observatory.
- **Редактор маршрутизации (меню ⋮):** правила (добавить / править / удалить /
  переупорядочить — inbound / домен / IP / порт / источник / сеть / протокол /
  пользователь → outbound или балансировщик) и балансировщики (тег / стратегия /
  селектор / fallback), плюс стратегия маршрутизации.
- **Редактор DNS (меню ⋮):** включение, DNS-серверы (строкой или объектом — порт /
  стратегия запросов / домены / ожидаемые и неожиданные IP / skip-fallback / …),
  пулы FakeDNS и опции уровня DNS.
- **General / Логи (меню ⋮):** стратегия маршрутизации, уровни логов (access /
  error / mask / DNS-лог), URL проверки outbound и переключатели статистики.
- Все четыре раздела конфига Xray сливаются в один конфиг (соседние и незнакомые
  ключи сохраняются), только при входе по логину.

## [0.3.16] — 2026-06-10

### Добавлено
- **Иконка приложения:** своя adaptive-иконка (индиго-монограмма «3X» с
