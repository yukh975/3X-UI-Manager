# 3X-UI Manager — приложение для Android

Нативный Android-клиент для управления панелью [3x-ui](https://github.com/MHSanaei/3x-ui) через REST API — дашборд, входящие, клиенты (с QR), узлы и конфиг Xray, на ходу.

## Возможности

### Подключение
- Два режима аутентификации (выбор на экране подключения): **API-токен** (Bearer) или **логин + пароль** (с опциональной 2FA).
- Переключатель **самоподписанного TLS** для панелей с self-signed сертификатом.
- Поле **Базовый URL подписки** (см. [Подписки](#подписки)).
- Данные хранятся в зашифрованном виде (`EncryptedSharedPreferences`, AES-256, ключ в Android Keystore).

### Дашборд
- Состояние сервера в реальном времени, опрос каждые 3 с: **Xray** работает/остановлен + версия, перезапуск Xray в один тап (с подтверждением).
- **CPU**, **Память**, **Диск** с занято/всего; **Онлайн** клиентов, **Сеть ↑/↓** в секунду, число соединений **TCP/UDP**, средняя нагрузка, аптайм, публичный IP.
- Тап по любой карточке метрики (CPU, Память, Диск, Нагрузка, Сеть, Соединения) → **график истории** с выпадающим выбором интервала (от реального времени до 5 ч).
- **Трафик за месяц** — проксированный (VPN) трафик за текущий месяц по собственным inbound'ам основной панели (с датой начала периода).
- Тап по **Онлайн** → список подключённых клиентов с указанием их inbound (обновляется каждые 3 с).
- Карточка **версии панели 3x-ui** внизу; если есть обновление — кнопка **Обновить** (с подтверждением) запускает самообновление панели.
- **Гео-базы** — пофайловое обновление встроенных гео-баз панели (`geoip.dat`, `geosite.dat` и RU/IR-варианты); после каждого обновления панель перезапускает Xray (с подтверждением).

### Входящие (Inbounds)
- Список с переключателем **вкл/выкл** по строке, трафик относительно лимита, число клиентов.
- **Создание / изменение / удаление** структурным редактором (без сырого JSON для типовых случаев):
  - Основное: примечание, порт, listen IP, протокол, включён.
  - Лимиты: лимит трафика (ГБ), расписание сброса, срок действия (выбор даты).
  - **Транспорт**: сеть (tcp/ws/grpc/httpupgrade/xhttp/kcp) с нужными полями (ws/httpupgrade — path+host, grpc — service name).
  - **Безопасность**: none / TLS (SNI) / Reality (dest, server names, short IDs, fingerprint, public/private key).
  - **Sniffing**: включение + чекбоксы destOverride.
  - **Advanced (JSON)**: protocol-`settings` (decryption, fallbacks, …). Список клиентов здесь **не** показывается — клиентами управляет вкладка Clients, и при сохранении inbound они не затрагиваются.

### Клиенты
- Список с индикатором онлайна, трафиком, сроком, «последний онлайн».
- **Поиск по email** — фильтрация списка по части имени/адреса.
- **Создание / изменение / удаление** (email/имя редактируется; переименование поддерживается): привязка к inbound (мультивыбор; attach/detach при сохранении), лимит трафика, лимит IP, период сброса, срок, Telegram ID, группа, комментарий.
- **Шит «Поделиться»** (тап по клиенту): секция **Подписка** (QR + ссылка) и разворачиваемая секция **Соединения**, где у каждой серверной ссылки свой QR + копировать/поделиться.

### Узлы (Nodes)
- Управление удалёнными панелями (мульти-панель): статус онлайн, CPU/RAM/задержка, число inbound/клиентов, **трафик за месяц** по каждому узлу.
- **Добавление / изменение / удаление**: имя, адрес, порт, схема, базовый путь, API-токен, режим проверки TLS, разрешение приватных адресов.

### Конфиг Xray (меню ⋮)
Структурные редакторы конфига Xray — каждая правка сливается в **весь** конфиг, сохраняя соседние и незнакомые ключи. Набор полей повторяет формы самой панели. (Нужен доступ по API-токену на панели **v3.3.0+**, иначе вход по логину.)
- **Outbounds** — список (добавить / править / удалить / переупорядочить) + формы по протоколам (vless, vmess, trojan, shadowsocks, socks, http, freedom, blackhole, wireguard) с транспортом и TLS/REALITY; **импорт из `vless://`-ссылки**.
- **Маршрутизация** — правила (источник / назначение / inbound → outbound или балансировщик, с переупорядочиванием) + балансировщики (стратегия / селектор / fallback) + стратегия маршрутизации.
- **DNS** — включение, опции уровня DNS, серверы (строкой или объектом), пулы FakeDNS.
- **General / Логи** — уровни логов, стратегия маршрутизации, URL проверки outbound, переключатели статистики.
- **Конфиг Xray (сырой)** — весь конфиг как JSON, запасной вариант для того, что не покрыто формами (Observatory, продвинутый xHTTP, hysteria, reverse, …).

### Бэкап / восстановление (меню ⋮)
- **Сделать бэкап** всей базы панели (настройки, inbounds, клиенты **и** конфиг Xray) в файл через системный выбор файлов и **восстановить** панель из него. Не зависит от движка — панель сохраняет SQLite как `x-ui.db`, PostgreSQL как `x-ui.dump`, и принимает любой обратно. Восстановление сначала спрашивает подтверждение и перезапускает Xray (кратко рвёт соединения). Работает по API-токену.

### Администрирование панели (меню ⋮)
- **Учётная запись админа** — смена логина и пароля входа в панель (для подтверждения нужны текущие данные).
- **API-токены** — список, создание (плейнтекст показывается один раз для копирования), вкл/выкл, удаление.
- **Перезапуск панели** — рестарт службы панели (с подтверждением; приложение переподключается через несколько секунд).

### Прочее
- **Язык**: английский (по умолчанию) или русский, переключение в **Настройках** (меню ⋮) — без перезапуска.
- Окно подтверждения на каждое **сохранение** и **удаление**, чтобы ничего не изменилось случайно.

## Аутентификация

| Режим | Что работает |
|------|-----------|
| **API-токен** (Bearer) | На панели **v3.3.0+** — всё, что умеет приложение: дашборд, inbounds, клиенты, узлы, **редактор конфига Xray**, **настройки**, **ссылки подписки** и **бэкап / восстановление**. Токен не протухает по времени, поэтому не выкидывает посреди работы. |
| **Логин + пароль** (сессия, опц. 2FA) | То же самое. Нужен только для панелей **старше v3.3.0**, где настройки и конфиг Xray отдаются сессии, но не токену. |

На панели **v3.3.0** весь управляющий API переехал под `/panel/api/*`, который авторизует Bearer-токен (как первого админа панели). Поэтому токен теперь покрывает и **настройки**, и **редактор конфига Xray**, и **авто-определение базы подписки** — то, что раньше требовало сессии. Токен 3x-ui — это **полный админ** (read-only и scoped-токенов нет), так что берегите его как пароль.

Токен создаётся в панели: **Settings → Security → API Token**.

**Чем удобен токен:** **логин**-сессия живёт `sessionMaxAge` (по умолчанию **360 мин / 6 ч**, Settings → Security) и затем отваливается — приложение переавторизуется по сохранённым логину/паролю, что **снова запрашивает 2FA**, если она включена. **Токен по времени не протухает**, поэтому не выкидывает и не просит 2FA повторно.

Единственный краевой случай: если токен потом **отключат или пересоздадут** в панели, его запросы начнут падать с `401`. Приложение умеет тихо переавторизовать только сохранённый **логин/пароль** профиль (но не токен), поэтому соединение по токену просто покажет ошибку, и вы переподключитесь — на практике редкость, ведь токены не истекают по таймеру.

## Подписки

**Ссылка-подписка** — это один URL, который отдаёт клиентскому приложению *все* конфиги пользователя и держит их актуальными. В 3x-ui она не хранится — она вычисляется как:

```
<база sub-сервера>/<subId клиента>
```

`subId` уникален у каждого клиента (приложение его уже знает), поэтому у каждого клиента своя ссылка. Общая часть — только **база sub-сервера**, и это настройка панели.

- **Авто (панель v3.3.0+):** приложение само читает базу из панели — **и по токену, и по логину** — QR и ссылка подписки появляются по каждому клиенту без настройки.
- **Ручная замена:** задать базу **один раз** в поле **Базовый URL подписки** на экране подключения, когда авто-чтение не подходит — панель **старше v3.3.0** (её настройки токену недоступны) или **обратный прокси** перед подпиской, чей публичный URI отличается от хранимого в панели. Укажите:
  - ваш **URI обратного прокси** (напр. `https://sub.example.com/`), либо
  - **Subscription URL** панели как он показан в панели.

  Дальше приложение строит ссылку каждого клиента как `база + subId`.

**Где взять в панели:** *Settings → Subscription* — включение, порт, путь и домен. Полная база обычно вида `https://ваш-хост:2096/sub/` (порт/путь — те, что вы настроили). Если перед подпиской стоит обратный прокси — используйте его публичный URI.

## Стек

- Kotlin 2.1 + Jetpack Compose (Material 3)
- Min SDK 24 (Android 7.0), target/compile SDK 35 (Android 15)
- Hilt (DI), Retrofit + OkHttp + kotlinx.serialization (сеть), zxing (QR-коды)
- Внутренняя локализация через лёгкий `tr()` (английский источник + русский словарь)

## Сборка

Откройте папку `android/` в Android Studio — она синхронизирует и скачает всё, включая Gradle wrapper.

```bash
cd android
gradle wrapper --gradle-version 8.10.2   # один раз
./gradlew assembleDebug                   # debug APK
./gradlew assembleRelease                 # подписанный release APK (нужен keystore)
./gradlew testDebugUnitTest               # юнит-тесты
```

## CI / релизы

[`.gitlab-ci.yml`](../.gitlab-ci.yml) собирает **только** по тегу версии (`vX.Y.Z`) или ручному запуску — пуши в ветку сборку не запускают. Пайплайн по тегу собирает **подписанный release APK**, прогоняет юнит-тесты, загружает APK в Generic Package Registry и автоматически создаёт GitLab Release. `versionName`/`versionCode` берутся из тега.

Чтобы выпустить релиз: запушьте тег `vX.Y.Z`.
