# Список изменений

Все значимые изменения приложения 3X-UI Manager для Android.
Формат основан на [Keep a Changelog](https://keepachangelog.com/), версии — по
[семантическому версионированию](https://semver.org/).

🇬🇧 [English version](CHANGELOG.md)

## [0.3.3] — 2026-06-08

### Исправлено
- Диалоги редакторов (клиент, inbound, node, Xray) теперь занимают всю
  высоту экрана. Раньше они подгонялись под содержимое, поэтому длинные
  формы выходили за экран: контент не прокручивался, а низ (кнопка Delete /
  последнее поле) обрезался. Предыдущая правка инсетов это не решала —
  настоящей причиной была подгонка высоты диалога под контент.

## [0.3.2] — 2026-06-08

### Изменено
- Редактор inbound'а теперь структурный вместо сырого JSON: транспорт
  (сеть + поля ws/grpc/httpupgrade), безопасность (SNI для TLS; для Reality —
  dest/serverNames/shortIds/fingerprint/ключи) и sniffing (переключатель +
  чекбоксы) редактируются дропдаунами, переключателями и полями.
  Неотмоделированные ключи конфига сохраняются. Protocol `settings`
  (decryption/fallbacks/…) остаётся в разделе Advanced (JSON).

### Исправлено
- Редактор inbound'а больше не показывает список клиентов inbound'а.
  Клиентами управляет вкладка Clients, и при сохранении inbound'а они
  остаются нетронутыми.

## [0.3.1] — 2026-06-08

### Добавлено
- Email/имя клиента теперь редактируется при изменении (переименование).
- Привязка клиента к inbound'ам редактируется при изменении (attach/detach
  по разнице).
- Опциональное поле «Subscription base URL» на экране подключения —
  включает ссылку+QR подписки по каждому клиенту в режиме API-токена
  (токену недоступны настройки подписки панели).
- Окно подтверждения на каждое сохранение, чтобы ничего не менялось
  случайно. Удаления уже подтверждались ранее. (Переключатель включения/
  выключения inbound'а остаётся в один тап — он быстрый и обратимый.)

### Исправлено
- Диалоги редакторов больше не уходят под навигационную панель Android —
  кнопка «Delete» и последнее поле формы снова видны полностью.
- Экранная клавиатура больше не перекрывает активное поле в редакторах.

## [0.3.0] — 2026-06-08

### Добавлено
- **Редактор клиентов** — создание, изменение и удаление клиентов: email,
  привязка к inbound'ам, лимит трафика, лимит IP, период сброса, срок действия
  (выбор даты), Telegram ID, группа, комментарий.
- **Редактор inbound'ов** — создание, изменение и удаление: имя, порт, listen,
  протокол, лимит трафика, расписание сброса, срок, а также сырой JSON
  `settings` / `streamSettings` / `sniffing` для любого транспорта/TLS/Reality.
- **Вкладка Nodes** — управление удалёнными панелями (мульти-панель): список со
  статусом онлайн, CPU/RAM/задержкой/счётчиками; добавление, изменение,
  удаление.
- **Редактор конфига Xray** — правка полного конфига Xray (outbounds, routing,
  DNS) как JSON. Требует входа по логину/паролю (панель отдаёт его только
  сессии).
- Ссылка-подписка в шит-меню клиента, собирается из настроек подписки панели
  (режим логин/пароль).

### Изменено
- Шит-меню клиента переделано в аккордеон: секция **Subscription** (QR +
  ссылка) и секция **Connections**, где каждая ссылка раскрывается в свой QR +
  копировать/поделиться.

### Исправлено
- Кнопки действий в шит-меню сжимались в нерабочую тонкую полосу, когда у
  клиента было несколько ссылок-соединений.

## [0.2.2] — 2026-06-07

### Исправлено
- DTO приведены в соответствие реальным ответам API 3x-ui v3.x — устранена
  ошибка «Unexpected response» при подключении: вложенные объекты
  `mem`/`xray`/`netIO` в статусе сервера, числовой `tgId` и вложенный `traffic`
  у клиентов, ответ ссылок клиента в виде массива.
- Поток логин/пароль трактует ответы-подтверждения `{success, obj:null}` как
  успех, а не как ошибку.

## [0.2.0] — 2026-06-07

### Добавлено
- Дашборд с опросом состояния сервера в реальном времени.
- Список inbound'ов с переключателем вкл/выкл и статистикой трафика.
- Список клиентов со ссылкой-подпиской и QR-кодом.
- Нижняя навигация: Dashboard / Inbounds / Clients.

## [0.1.0] — 2026-06-07

### Добавлено
- Первая устанавливаемая сборка.
- Экран подключения с двумя режимами аутентификации: API-токен или
  логин/пароль с опциональным 2FA. Переключатель самоподписанного TLS.
- Каркас проекта, GitLab CI, пайплайн подписанного релиза с версионированием
  из git-тегов.
