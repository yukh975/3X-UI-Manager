# Список изменений

Все значимые изменения приложения 3X-UI Manager для Android.
Формат основан на [Keep a Changelog](https://keepachangelog.com/), версии — по
[семантическому версионированию](https://semver.org/).

🇬🇧 [English version](CHANGELOG.md)

## [0.3.15] — Не выпущено

### Добавлено
- **Дашборд:** тап по любой карточке системной метрики (CPU, Память, Диск,
  Нагрузка, Сеть, Соединения) открывает **график истории** этой метрики с выбором
  интервала сверху (Реальное время / 30 мин / 1 час / 2 / 3 / 5 часов; по
  умолчанию **1 час**). Блоки с несколькими значениями рисуют все линии: Нагрузка —
  1/5/15 мин, Сеть — ↑/↓, Соединения — TCP/UDP. Данные берутся из
  `GET /panel/api/server/history/{metric}/{bucket}` панели (по Bearer-токену),
  ~60 точек на интервал. Процентные метрики — шкала 0–100 %, остальные
  масштабируются автоматически.

## [0.3.14] — 2026-06-09

### Добавлено
- **Дашборд:** карточка «Трафик за месяц» — проксированный (VPN) трафик за
  текущий месяц по **собственным inbound'ам основной панели** (без суб-узлов),
  с датой начала периода («с 01.06»). Считается на стороне приложения
  группировкой `/inbounds/list` по `nodeId` и суммированием up+down — без
  дополнительных запросов к панели.
- **Узлы:** в карточке каждого узла теперь виден **трафик за месяц** (по
  inbound'ам этого узла). Звёздочка `*` отмечает группу, где не все inbound'ы
  сбрасываются ежемесячно (тогда значение считается за всё время, а не за месяц).
- **Дашборд:** карточка «Гео-базы» — пофайловое обновление встроенных гео-баз
  панели: `geoip.dat`, `geosite.dat` и RU/IR-варианты
  (`POST /panel/api/server/updateGeofile/:file`). Каждое обновление с
  подтверждением, т.к. панель после него перезапускает Xray (кратко рвёт связь).

### Изменено
- **Совместимость с панелью v3.3.0:** в апстриме v3.3.0 эндпоинты конфига Xray и
  настроек панели переехали под `/panel/api/*`. `XuiApi` обновлён под новые пути —
  и так как там они принимают Bearer-токен, **редактор конфига Xray** и
  авто-**ссылка подписки** теперь работают и в режиме **API-токена** (раньше —
  только логин/пароль). ⚠️ Требуется панель **v3.3.0+**.

### Исправлено
- **Онлайн по серверам:** в группе «основной сервер» больше не показываются
  клиенты, принадлежащие только узлам. Центральный эндпоинт онлайна отдаёт
  онлайн-почты по всему дереву узлов, поэтому в «основном» мог оказаться клиент,
  состоящий только в инбаунде узла (например, учётка outbound, которой основной
  сервер цепляется к узлу). Теперь «основной» сервер показывает только клиентов
  собственных инбаундов панели (`nodeId` 0); клиенты узлов — под своим узлом.
  Если узел недоступен напрямую с устройства, его онлайн-клиенты теперь
  определяются по принадлежности к инбаундам, а не пропадают из списка.

## [0.3.12] — 2026-06-09

### Изменено
- **Дашборд:** каждый показатель теперь отдельной строкой на всю ширину, по
  порядку — CPU, Память, Диск, Нагрузка, Сеть, Соединения, Онлайн (раньше
  Нагрузка/Сеть/Соединения/Онлайн были тесными плитками в половину ширины).
- **Узлы:** карточка узла переделана — после URL каждый факт на своей строке:
  CPU · RAM, **Пинг** (значение в ms теперь подписано), inbounds · клиенты,
  аптайм и версия 3x-ui узла.
- **Клиенты:** «Истекает» теперь показывает **количество оставшихся дней** (а не
  дату) и с двоеточием; «Был онлайн» вынесен на отдельную строку, тоже с
  двоеточием, и для тех, кто ни разу не заходил, пишет «Никогда» (а не прочерк).
- **Inbounds:** двоеточие после «Истекает».

### Добавлено
- **Узлы:** если на узле 3x-ui старее последней версии, появляется кнопка
  **«Обновить»** — запускает самообновление узла через центральную панель
  (`POST /panel/api/nodes/updatePanel`).
- **Релиз:** имя публикуемого APK теперь содержит номер версии
  (`3x-ui-manager-<версия>.apk`).

## [0.3.11] — 2026-06-08

### Исправлено
- После **Старт/Стоп/Перезапуск Xray** карточка больше не «моргает»: раньше
  сразу после подтверждения кнопка на миг показывала новое состояние, потом
  возвращалась к старому и только через ~3 с (следующий опрос) показывала
  правильное. Причина — панель ещё долю секунды отдаёт старое состояние Xray, и
  немедленный опрос перетирал оптимистичное. Теперь ожидаемое состояние
  «закрепляется» на несколько секунд (пока Xray реально не переключится), а
  дашборд обновляется сразу — без отката и без задержки.

## [0.3.10] — 2026-06-08

### Изменено
- На дашборд **возвращены кнопки «Запустить» / «Остановить» Xray** (для всех
  типов авторизации — и токен, и логин/пароль), вместе с «Перезапустить».
  Потеря связи при остановке Xray, судя по всему, происходит только когда сама
  панель проксируется через Xray; при прямом подключении остановка безопасна.
  После действия карточка сразу показывает корректную кнопку (оптимистичное
  состояние), даже если опрос не вернулся.

## [0.3.9] — 2026-06-08

### Добавлено
- **Онлайн-клиенты по серверам.** Тап по «Онлайн» на дашборде теперь показывает
  список, сгруппированный по серверам: сначала основной сервер, затем каждая
  нода со своими подключёнными клиентами. Каждая нода опрашивается напрямую по
  своему API (адрес + токен из списка нод), поэтому видно реальную картину —
  включая клиента, подключённого сразу к нескольким нодам (чего центральный API
  показать не мог). Ноды опрашиваются параллельно.
- **Аптайм Xray** на карточке состояния (время с последнего перезапуска Xray,
  отдельно от общего аптайма сервера).

### Изменено
- На дашборде оставлена только кнопка **«Перезапустить» Xray**; «Запустить» и
  «Остановить» убраны полностью (для всех типов авторизации). Остановка Xray на
  панелях, проксируемых через Xray, рвёт доступ к самой панели — подтверждено и
  с токеном, и с логином/паролем; «Перезапустить» безопасен.
- На экране подключения переключатель режима укорочён до **«API Token» / «Login»**
  (было «Login & password» — длинный текст переносился и ломал вёрстку).

## [0.3.8] — 2026-06-08

### Изменено
- Управление **Xray на дашборде зависит от типа авторизации**: с **API-токеном**
  доступен только **«Перезапустить»**; по **логину/паролю** — полный набор
  **«Запустить» / «Остановить» / «Перезапустить»**. Причина: на конфигурациях,
  где панель проксируется через Xray, остановка Xray рвёт доступ к панели/API, и
  токен-сессия не может его поднять обратно. «Перезапустить» безопасен — Xray
  поднимается, и связь восстанавливается.
- В списке **онлайн-клиентов** показываются только email (без inbound): 3x-ui
  учитывает онлайн/трафик по email под одним каноническим inbound и копирует эту
  запись в остальные, поэтому определить реальные «живые» inbound'ы клиента через
  API невозможно — чтобы не вводить в заблуждение, inbound убран.

## [0.3.7] — 2026-06-08

### Изменено
- Слова **inbound/outbound** в русском интерфейсе больше не переводятся
  («Изменить inbound», «Удалить inbound» и т.д.) — это устоявшиеся термины.
- В шит-меню клиента кнопки **Изменить** и **Удалить** поменяны местами
  (Изменить слева, Удалить справа), а **Закрыть** теперь полноценная кнопка,
  а не ссылка.
- Кнопки управления Xray на дашборде переносятся на вторую строку по центру,
  если не помещаются в одну (например, «Перезапустить» + «Остановить» в RU).

### Исправлено
- После **остановки Xray** дашборд сразу показывает кнопку «Запустить», даже
  если связь с панелью пропала (на конфигурациях, где панель проксируется через
  Xray, опрос после остановки может не вернуться) — раньше оставались неверные
  кнопки «Перезапустить/Остановить».
- На экране **конфига Xray** в режиме API-токена больше не показывается
  «Network error» — достаточно пояснения, что нужен вход по логину/паролю.

## [0.3.6] — 2026-06-08

> Все правки этого выпуска проверены на эмуляторе (скриншоты редакторов и
> настроек), а не только компиляцией.

### Исправлено
- **Список онлайн-клиентов** теперь определяет inbound по `clientStat.inboundId`
  (тот, под которым панель реально учитывает клиента), а не по тому, в чьём
  списке клиент перечислен. Один email может быть участником нескольких
  inbound'ов (один пользователь на нескольких серверах) — 3x-ui хранит его
  трафик/онлайн под одним каноническим inbound и дублирует эту запись в
  остальных, поэтому раньше показывались все. Теперь — только тот, к которому
  клиент привязан панелью.
- **Кнопки «Удалить» (inbound/node) больше не уходят за край экрана под
  системное меню.** Редакторы теперь рисуются полноэкранно в окне приложения
  (а не в `Dialog`, где системные отступы в Compose не приходили), а кнопка
  удаления закреплена в нижней панели и всегда видна — скроллить к ней не нужно.
- **Поле «Комментарий» в редакторе клиента больше не перекрывается
  клавиатурой** — форма поднимается над клавиатурой, поле доступно.
- **Переключатели и кнопки в Настройках и редакторах на русском больше не
  наезжают на текст** — длинный текст получает своё место (weight), элемент
  управления остаётся справа.

## [0.3.5] — 2026-06-08

### Добавлено
- **Обновление дашборда жестом** — потяните страницу вниз сверху, чтобы вручную
  обновить состояние сервера, список онлайна и проверку обновления панели.
- **Управление Xray по состоянию** на карточке статуса: если запущен — кнопки
  **Перезапустить** и **Остановить** (обе с подтверждением); если остановлен —
  только **Запустить**.

### Исправлено
- **Список онлайн-клиентов** теперь показывает только те inbound'ы, к которым
  клиент подключён прямо сейчас (по свежести `lastOnline` — клиент может быть
  подключён к нескольким одновременно), а не все inbound'ы, в которые он добавлен.
- **Кнопки удаления** в редакторах inbound и node больше не прячутся под
  системной навигацией Android, а поле **Комментарий** в редакторе клиента
  больше не перекрывается клавиатурой — диалоги-редакторы теперь корректно
  учитывают системные отступы и сжимаются под клавиатуру.

### Изменено
- **Полный перевод интерфейса** на русский: дашборд, входящие, клиенты, узлы,
  все редакторы, конфиг Xray, шит «Поделиться» и экран подключения — раньше
  переводилось только меню.

## [0.3.4] — 2026-06-08

### Добавлено
- **Поиск клиентов по email** — фильтрация списка по части имени/адреса.
- **Дашборд → Онлайн**: тап по числу онлайн открывает список подключённых
  клиентов с указанием их inbound (обновляется каждые 3 с).
- **Версия Xray** на карточке состояния и **карточка версии панели 3x-ui**
  внизу дашборда; при наличии обновления — кнопка «Обновить» (с подтверждением)
  запускает самообновление панели.
- **Переключение языка RU/EN** прямо в приложении (Настройки, меню ⋮) — без
  перезапуска.
- **Страница «Настройки»** с разделом «О приложении» (версия, копирайт).
- **Сохранение сессии**: авто-релогин при запуске и при ответе 401 — после
  входа по логину/паролю приложение больше не выкидывает.
- **Блокировка приложения**: код-пароль на запуск + разблокировка по отпечатку
  пальца (биометрия).

### Изменено
- Релизный APK теперь называется `3x-ui-manager-release.apk` (вместо
  `app-release.apk`) — созвучно названию проекта.
- CI больше не собирает debug-APK — собирается только подписанный release.
- Страница релизов и заметки к релизу теперь на русском и формируются из
  этого чейнджлога (раздел соответствующей версии).

### Исправлено
- Экран блокировки больше не падал на компиляции: метки запроса биометрии
  выносятся в composable-scope (`tr()` нельзя вызывать из обычной функции).

## [0.3.3] — 2026-06-08

### Исправлено
- Диалоги редакторов (клиент, inbound, node, Xray) теперь занимают всю
  высоту экрана. Раньше они подгонялись под содержимое, поэтому длинные
  формы выходили за экран: контент не прокручивался, а низ (кнопка Delete /
  последнее поле) обрезался. Предыдущая правка инсетов это не решала —
  настоящей причиной была подгонка высоты диалога под контент.

## [0.3.2] — 2026-06-08

### Изменено
- Редактор inbound'а теперь структурный вместо сырого JSON: транспорт
  (сеть + поля ws/grpc/httpupgrade), безопасность (SNI для TLS; для Reality —
  dest/serverNames/shortIds/fingerprint/ключи) и sniffing (переключатель +
  чекбоксы) редактируются дропдаунами, переключателями и полями.
  Неотмоделированные ключи конфига сохраняются. Protocol `settings`
  (decryption/fallbacks/…) остаётся в разделе Advanced (JSON).

### Исправлено
- Редактор inbound'а больше не показывает список клиентов inbound'а.
  Клиентами управляет вкладка Clients, и при сохранении inbound'а они
  остаются нетронутыми.

## [0.3.1] — 2026-06-08

### Добавлено
- Email/имя клиента теперь редактируется при изменении (переименование).
- Привязка клиента к inbound'ам редактируется при изменении (attach/detach
  по разнице).
- Опциональное поле «Subscription base URL» на экране подключения —
  включает ссылку+QR подписки по каждому клиенту в режиме API-токена
  (токену недоступны настройки подписки панели).
- Окно подтверждения на каждое сохранение, чтобы ничего не менялось
  случайно. Удаления уже подтверждались ранее. (Переключатель включения/
  выключения inbound'а остаётся в один тап — он быстрый и обратимый.)

### Исправлено
- Диалоги редакторов больше не уходят под навигационную панель Android —
  кнопка «Delete» и последнее поле формы снова видны полностью.
- Экранная клавиатура больше не перекрывает активное поле в редакторах.

## [0.3.0] — 2026-06-08

### Добавлено
- **Редактор клиентов** — создание, изменение и удаление клиентов: email,
  привязка к inbound'ам, лимит трафика, лимит IP, период сброса, срок действия
  (выбор даты), Telegram ID, группа, комментарий.
- **Редактор inbound'ов** — создание, изменение и удаление: имя, порт, listen,
  протокол, лимит трафика, расписание сброса, срок, а также сырой JSON
  `settings` / `streamSettings` / `sniffing` для любого транспорта/TLS/Reality.
- **Вкладка Nodes** — управление удалёнными панелями (мульти-панель): список со
  статусом онлайн, CPU/RAM/задержкой/счётчиками; добавление, изменение,
  удаление.
- **Редактор конфига Xray** — правка полного конфига Xray (outbounds, routing,
  DNS) как JSON. Требует входа по логину/паролю (панель отдаёт его только
  сессии).
- Ссылка-подписка в шит-меню клиента, собирается из настроек подписки панели
  (режим логин/пароль).

### Изменено
- Шит-меню клиента переделано в аккордеон: секция **Subscription** (QR +
  ссылка) и секция **Connections**, где каждая ссылка раскрывается в свой QR +
  копировать/поделиться.

### Исправлено
- Кнопки действий в шит-меню сжимались в нерабочую тонкую полосу, когда у
  клиента было несколько ссылок-соединений.

## [0.2.2] — 2026-06-07

### Исправлено
- DTO приведены в соответствие реальным ответам API 3x-ui v3.x — устранена
  ошибка «Unexpected response» при подключении: вложенные объекты
